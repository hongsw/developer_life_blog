Elixir regex — http, email, img, tel, address

Elixre http://www.elixre.uk/

  1. url

    title = “hello http://medium.com/?test=11 12”

    #
    regex_url = ~r/(https?:\/\/)?(([\da-z.-]+).([a-z.]{2,6}))([\/\w .-])\/?(\&?)(([?])([^ \t\r\n\v\f])*)/

    첫번째 매칭되는 첫번째 문장

    link = Regex.run(regex_url, title) |> hd

    Replace matches in a string

    title = Regex.replace(regex_url, title, “”)

5. address (작성중)

[-a-zA-Z0–9:%_+.~#?&//=]를 넣으면 구문에러가 발생한다.

[^ \t\r\n\v\f]으로 변경하여 처리하였다.

Elixir로 15분안에 블로그를 만들자

Study0730 모임 3번째 주제

어제 스터디를 하이브아레나에서 했습니다.

저는 Jakub Cieslar가 쓴 Elixir blog in 15 minutes using Phoenix framework — Step by Step 을 같이 진행해보았습니다.

확인해볼 사안은 2015년 Aug에 작성된 글이라서 지금이랑 뭐가 따른지 혹은 더 개선할것들은 없는지 확인해보면서 기본적인 투토리얼을 최신버전으로 정리하고자 합니다.

  1. 저자가 git에 공개한 페이지를 가져와서 하는 방법도 있지만,저자의 의도대로 mix (phoenix build tool) 로 하나하나 만드는 것을 따라해보겠습니다.

  2. 우선 준비할것들 linux or mac을 준비한상태에서 ASDF(언어 및 데이터베이스 등 개발자를 위한 설치 및 버전관리 툴)를 설치하고, ASDF를 이용해서 erlang과 elixir까지 설치 한다. https://www.icicletech.com/blog/elixir-and-erlang-setup-with-asdf-version-manager

  3. Phoenix Framework 도 설치 합니다.

    $ mix archive.install [https://github.com/phoenixframework/archives/raw/master/phoenix_new.ez](https://github.com/phoenixframework/archives/raw/master/phoenix_new.ez)

  4. 프로젝트를 만듭니다. 관련된 기본적인 dependencies를 설치 하면 세팅완료되고, 프로젝트의 소스를 git으로 관리 합니다.

    $ mix phoenix.new elixir_phoenix_blog Fetch and install dependencies? [Yn] Y

    $ cd elixir_phoenix_blog
    $ git init

  5. 데이터베이스는 기본적으로 postgres를 사용하도록 세팅되고 mysql은 설정을 변경해야합니다.

    $ vi config/dev.exs

    Configure your database

    config :elixir_phoenix_blog, ElixirPhoenixBlog.Repo,

    adapter: Ecto.Adapters.**MySQL**,
**username: "mysql_username",
password: "mysql_password",
database: "elixir_phoenix_blog",
hostname: "localhost",
port: 3306,**
pool_size: 10

    end of vi

    $ vi mix.exs # Configuration for the OTP application.

    Type mix help compile.app for more information.

    def application do
    [mod: {ElixirPhoenixCircleciBlogStarter, []}, 

    applications: [:phoenix, :phoenix_pubsub, :phoenix_html, 
:cowboy, :logger, :gettext, 
:phoenix_ecto, :**mariaex**]]

    end

    …

    defp deps do 

    [{:phoenix, "~> 1.2.1"}, 
{:phoenix_pubsub, "~> 1.0"}, 
{:phoenix_ecto, "~> 3.0"}, 
{:postgrex, ">= 0.0.0"}, 
{:phoenix_html, "~> 2.6"}, 
{:phoenix_live_reload, "~> 1.0", only: :dev}, 
{:gettext, "~> 0.11"}, 
{:cowboy, "~> 1.0"}**, 
{:mariaex, "~> 0.7"}**]

    end

    mariaex is the Ecto recommended MySQL driver.

    end of vi

    $ mix deps.get # Ecto driver 가져오기

    $ mix ecto.create

    phoenix이 CRUD을 위해 contorol,model,view(html)를 자동으로 web폴더에 생성해준다.**

    $ mix phoenix.gen.html Post posts title:string body:text**

    controller 를 라우터 파일에 넣으면 라우팅이 된다.

    $ vi web/router.ex # add resources “/posts”, PostController
    scope “/“, ElixirPhoenixCircleciBlogStarter do 

    pipe_through :browser 
# Use the default browser stack 
get "/", PageController, :index

    resources “/posts”, PostController end

    end of vi

    phoenix.gen 을 통해서 ecto.model 을 변경했으니 migrate한다 .

    $ mix ecto.migrate Compiling 8 files (.ex)
    11:35:43.432 [info] == Running ElixirPhoenixBlog.Repo.Migrations.CreatePost.change/0 forward
    11:35:43.432 [info] create table posts
    11:35:43.625 [info] == Migrated in 0.1s

    성공적!

  6. 서버를 실행시키고 첫번째 포스트를 올려봅시다.

    $ mix phoenix.server

http://localhost:4000/posts 에서 새글을 넣어본 상태

여기 까지의 파일을 올려두었으니 연습하다 다른부분이 있으면 비교해보면 좋겠습니다.

개발자, MBA를 고민해보다

개발자에서 창업가로 가기 위한 길에 도움이 될까?

2011년에 창업하여 스타트업이라고 명명하고 여러 홈페이지를 대신 만들어주는 Web Agency를 운영하면서 자체 솔루션을 만들어왔었다.

6년이 지난 지금은 다양한 대신만들어주는 용역! 이라고 하는 일에 대한 노하우가 몸에 녹아졌으며 자체 솔루션은 시기가 지났거나 서비스화에 실패하였다.

최근 강제 고민이 가능한 시간 여유가 생겨서 원인과 해결책을 고민하기 시작하면서 나의 고민의 시작 MBA가 머리속에 떠다니기 시작했다.

“시장의 변화를 기술기반 사업아이템을 더욱 전문화하게 해야하지 않을까?”

고등교육(알고리즘 수업)을 받은 상황에서 MBA는 그다음 단계라고 생각했다.

하지만, 현실은 높은 수업료와 수업시간도 무시하지 못한다는 것이다. 잠시 검색해보니 해외에서도 NO PAY MBA로 홈페이지와 책도 출간한 사람이 있을 정도로 합리적인 가격으로 MBA 수업을 수료하고자 하는 사람들이 생긱고 있다.

정리해보자면, 가격별로 MBA를 정리해둔것이 있다.
MOOC MBA Design Archives - No-Pay MBA
When I started my MOOC MBA journey, I could never have predicted where this experiment in self-directed learning would…www.nopaymba.com

그리고 위의 그림처럼 Executive MBA 과정도 있는데 이것에 대한 비교도 만들었다. 역시 MBA는 비교테이블? ㅋ

어떤 수업내용일지 궁금함이 자연스럽게 생겼고 효과도 궁금했다.

결론적으로 아쉬운점만 본다면, 이런 수업은 초보에게 절대적 도움이 되는 피드백은 받을 수 없다는 점과, 미국의 현실에 맞춘 시장환경이 주된 문제점이고 위의 이미지 처럼 법률도 한국은 기대하기 어려운것 처음부터 알고 있었다고 생각해야겠다.

다시 생각해보자

정말도 돈을 내고 학위증을 받고 하는 것이 실력과 직결된다면 그다음에는 스스로 공부할 수 있을까? 중요한 피드백을 받는 것이 필수적이라면 수업이외에는 받는 것은 없을까? 한국현실과 현시점에 정확하게 필요한 수업을 찾는 것이 가능한 것일까?

  2. 일리노이대학에서 디지탈마케팅의 수업커리큘럼의 방향이 현 디지털 시대에 시점에서 필요하다 https://www.coursera.org/specializations/digital-marketing

  3. VC 없이 자기 사업을 하기 위한 강의 인 런던 비지니스 스쿨의 포인트도 필요하고 https://www.coursera.org/learn/startup-financing-without-vc

  4. 데이터 사이언스 수업도 부분적으로 필요하다면 더 추가하는 것도 좋을 거 같다. https://www.udemy.com/topic/data-science/

개인적인 일정과 연결하여 코스 전체 일정을 잡아보자

  1. 주 8시간을 코스 진행과정으로 잡고 관련서적을 리딩 보고서를 작성을 하는 것으로 코스별 시험을 대체한다.

프리랜서 연결 사이트의 문제점

나는 프리랜서 이지만, 나의 포트폴리오는 나의 것이다.

조심히 프로젝트 개발을 수락해야하고 이는 상호 심도있는 이해가 필요하다.(위험은 어디에나 있고 언제든지 나올수 있다.)

잘 몰랐다 왜 기존에 나와 있는 프리랜서 연결사이트들이 조금 기분이 이상한 상태로 프로젝트가 연결되기 위해서 갖은 노력을 했어야 하는지.

“썩 기분이 좋은 건 아니지만, 일은 줄 수도 있다고 하니 하라는 대로 할께~”

이런 생각으로 나의 포트폴리오, 경력, 작은 노하우 분야를 작성했다. 그러면서도 이렇게 적으면 그걸 이해하고(정확하게) 알아보고 물어볼 사람이 있을까? 라는 고민도 같이 하게 된다.

“처음뵙겠습니다. 어느분야 이신가요. 아 건설쪽에 있으시군요. 홈페이지는 집이라고 생각하시면 됩니다. 제가 홈페이지를 만드는 방법을 안내해드릴게요”

이렇게 듣는 사람=클라이언트의 경험이 어느쪽에 많은지에 따라서 설명하는사람=디벨로퍼는 조금의 설명이라도 그쪽에 맞추어 설명하게 되면 상호 이해도는 많이 올라 갈것 이라고 생각한다. 보통의 클라이언트는 자신의 분야를 다 일반적으로 개발자들이 알것이라고 생각하지는 않겠지만 서로 조금씩 상대방의 업(業)에 대해서 이해하고자 한다면 그 프로젝트는 성공적으로 끝날 가능성이 높다.

이러한 생각으로 두가지 가설을 세운다.

  1. 클라이언트는 자기 분야의 사람을 찾고자 한다.

  2. 디벨로퍼는 자기의 포트폴리오는 선별해서 보여주길 원한다.

위의 두가지 가설을 확인하기 위해서

  1. 기존 “개발언어” 중심의 프로젝트 태깅방식을 분야(예를 들어서, 공장자동화, 홈페이지, 검색, 크롤링, 등등) 으로 나누어 설명하고 개발언어 및 개발아키텍트와 무관하게 실사용자가 보고 구분 할 수 있는 업종별 기능단위로 명명한다.

  2. 디벨로퍼는 자기의 포트폴리오를 사진의 유무, 업종별기능 태그, 개발언어, 시기, 참여도, 관련OSS 등을 기록하고 프로젝트를 의뢰할 사람이 접촉시에 상호 동의에 의해서 포트폴리오를 의뢰자에 한해서 공개하는 것이다.

대표적 이미지 = 정확한 설명과 상호 도메인 이해가 산출물(결과물)에 미치는 영향에 대한 …

나는 싫었다.

  1. 하나의 프로젝트에 잘 알지도 못하지만 우선 프리랜서들이 줄서야 하는 잘 못된 구조

  2. 이로써 가격적 경쟁만 찍어내는 경쟁만 이루어 지는 시스템 구조

  3. 무슨일인지도 모르면서 프로젝트 만드는 클라이언트

디지털 노마드를 생활하는 외국 친구들을 부럽게 하이브아레나 커뮤니티에서 만나보면, 참 한국에서 아쉬움이 많다. 사다리로 계층을 올라갈 수 없다면 유목민으로라도 삶을 영위할 수 있도록 하는 유일한 수단이 프리랜서인데 정작 어떻게 계약서를 써야 하는지, 어떻게 일을 시작하고 끝내야 하는지 , 어떻게 요구사항을 명세할지, 어떻게 분쟁을 해결할지 를 아무도 알려주지 않은체 대학교 문제풀기 공장에서 졸업한 친구들은 어찌해야할것인가.

열린질문으로 글을 마무리 합니다.

실제로는 위의 사진처럼 이렇게 할수도 없고 (허리 아프고, 소금에 노트북쩌는 것도 싫고) 하지만, 우리가 원하는건 일하는 시간에는 일을 하고 쉬는시간에는 쉴 수 있는 (출퇴근시간이 아니라, 못난팀장설득시키는 회의가 아니라) 가족과 얼굴 볼 시간을 단 2시간이라도 더 벌려는 행위를 — 한국형 디지털 노마드 라고 해야겠다.
