일한 만큼의 보상, 그리고 DAPP

블록체인 커뮤니티의 전략적 변화 시기와 내가 관심을 가져야 하는 이유

7년간 작은 소프트웨어 개발 업체를 운영하면서 느낀것은 어떻게 하면은 우리가 일한 것에 대한 보상을 받을 것이가에 대한 해답을 찾는 과정 이였다.

다양한 경험을 둘로 나누면 보상을 받은 케이스와 못 받은 케이스로 나누어진다. (기타 법적인 절차로 해결하는건 추가 비용과 시간이 발생함으로 논외)

  • 지인의 소개로 일을 해줬는데 중간에 엎어 버리고 보상도 못 받기

  • 일을 정말 조금 해줬는데 기술을 누적된 효과로 보상을 정확하게 받기

우리는 보상을 잘~ 받은 것을 추구할 것이다.

쉽게 말해서 그냥 일 한 만큼 만이라도 리스크없이 보상을 받자는 것이 모든 이의 소망일 것이다. ( 기우제 없이 완벽한 날씨의 환경에서의 소작민이라고 생각해도 좋다)

보상을 못 받은 경우의 원인을 분석해보면

  1. 상호신뢰부족(일이 다 잘처리된것인지 상호 인지 부족)

  2. 비자동화된 잔금 프로세스(일 다해도 지연되어 받은경우)

  3. 클라이언트의 회사부도(일을 다해도 보상을 못받은 경우)

2가지가 가장 큰 문제점이고 클라이언트-개발사간의 가장 주된 이야기거리 이다.

정보의 격차와 입장차 때문에 생긴것이란건 잘 알고 있고 “개발사가 잘 설명을 해줘야 하는 것도 능력인것 맞지만, 의뢰자도 자알 일을 시켜야 한다.” (1번 상황)

또한, 경우에 따라선 서로 각분야에 잘 알고 있고 잘 이해하고 있는 의뢰자-개발자 의 관계로 만나도 해당 문제는 다양한 입장 때문에 생길 수 있다고 생각된다. (2,3번 상황)

정리하면, 모든 관계에서 작은 하나의 이해관계(체인) 이라도 문제가 생기면 기존 혹은 미래의 관계는 여지없이 불안해지고 어려워진다.

미래를 그려 본다, 블록체인 기술이 조금이나마 미래의 그 불안를 해소 할 수 있는 여러 방법중에 하나가 되길.

그리고 가장중요한건, 비용지출과 수입이라는 제로섬 관계에서 벗어나서 게임이론으로 무장된 블록체인으로 비제로섬인 행복의 섬으로 가고 싶은 것이 필자의 생각이다.

암호화폐들의 가격들이 악재로 모두 가격이 내려가면서 몇몇 좋은 혹은 재정적 능력이 있는 코인들은 그 코인의 네트워크 경제의 부흥과 경제성/유용성을 보여주기 위해 더욱 DAPP개발을 장려하려고 할 것이다. (마치 미국 대공황 시대에 뉴딜정책처럼)

이런 과정을 통해서 실생활 혹은 특정 환경에서 동작되는 새로운 유의미한 Application의 발견과 User interface 설계는 기존 개발자들과 기존 디자이너에게 도전과 기회를 제공할 것이다. (기존의 중앙집중방식의 방식에서 얻을 수 없는 응용 노하우가 필요하다)

Docker 를 이용한 워드프레스 개발 환경

프로젝트 마다 설정해주기 복잡하죠?

필수 개발 하드웨어 환경!

  1. MAC 구입한다.

  2. Windows PC는 버린다.

이하 자세한 설명은 정신건강을 위해서 생략한다.

개발 소프트웨어 환경!

  1. 개발가상환경을 구축하기 위해서 docker 를 설치 한다. https://www.docker.com/docker-mac

  2. mysql 접속을 위한 클라이언트로 sequelpro를 설치한다. http://www.sequelpro.com/

  3. 개발 에디터로 atom 를 사용한다. https://atom.io/

개발 폴더를 설정하고 개발 소스를 넣어서 가상환경에서 돌려보자 ~!

cd ~ && cd Github
mkdir myproject && cd myproject
git init
unzip myproject.zip

개발 폴더에 기존 레거시 스파게티 코드를 넣어주고 가상환경을 실행하자

모두 잘 알고 있듯이 워드프레스는 mysql, php이 기본적으로 필요합니다.

version: "2"
services:
  my-wpdb:
    image: mariadb
    volumes:
      - ./mysql:/var/lib/mysql 
    ports:
      - "8081:3306"
    environment:
      MYSQL_ROOT_PASSWORD: ChangeMeIfYouWant
  my-wp:
    image: wordpress
    volumes:
      - ./:/var/www/html
    ports:
      - "8088:80"
    links:
      - my-wpdb:mysql
    environment:
      WORDPRESS_DB_PASSWORD: ChangeMeIfYouWant

위의 내용을 docker-compose.yml 파일로 만들고 docker-compose up -d명령을 실행해서 나만의 개발환경을 시작하자( 처음에는 시간이 걸린다. )

http://localhost:8088/

Elixir regex — http, email, img, tel, address

Elixre http://www.elixre.uk/

  1. url

    title = “hello http://medium.com/?test=11 12”

    #
    regex_url = ~r/(https?:\/\/)?(([\da-z.-]+).([a-z.]{2,6}))([\/\w .-])\/?(\&?)(([?])([^ \t\r\n\v\f])*)/

    첫번째 매칭되는 첫번째 문장

    link = Regex.run(regex_url, title) |> hd

    Replace matches in a string

    title = Regex.replace(regex_url, title, “”)

2. email (작성중)

title = "hello abc@abc.com bye"

3. img (작성중)

title = "hello http://ab.com/logo.png bye"

4. tel (작성중)

title = "hello +82-10-9999-1111 bye"

5. address (작성중)

etc

원인은 모르겠지만 parameter 부분에

[-a-zA-Z0–9:%_+.~#?&//=]를 넣으면 구문에러가 발생한다.

[^ \t\r\n\v\f]으로 변경하여 처리하였다.

참조
정규 표현식 - 위키백과, 우리 모두의 백과사전
컴퓨터 과학의 정규 언어로부터 유래하였으나 구현체에 따라서 정규 언어보다 더 넓은 언어를 표현할 수 있는 경우도 있으며, 심지어 정규 표현식 자체의 문법도 여러 가지 존재하고 있다. 수많은 프로그래밍 언어가 정규…ko.wikipedia.org

*# A regex sigil to match 'foo'*
~r/foo/

*# Interpolation can be used*
~r/foo/ **=** ~r/#{"foo"}/

*# Test if a string matches*
true **=** Regex**.**match?( ~r/foo/ , "Hello foo")

*# Run returns first match*
["foo1", "1"] **=** Regex**.**run(~r/foo([0-9])/, "foo1 foo2")

*# Scan returns all matches*
[["foo1", "1"], ["foo2", "2"]] **=** Regex**.**scan(~r/foo([0-9])/, "foo1 foo2")

*# Replace matches in a string*
"one_two_three" **=** Regex**.**replace(~r/-/, "one-two-three", "_")
