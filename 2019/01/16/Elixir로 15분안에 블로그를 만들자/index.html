Elixir로 15분안에 블로그를 만들자

Elixir로 15분안에 블로그를 만들자

Study0730 모임 3번째 주제

어제 스터디를 하이브아레나에서 했습니다.

저는 Jakub Cieslar가 쓴 Elixir blog in 15 minutes using Phoenix framework — Step by Step 을 같이 진행해보았습니다.

확인해볼 사안은 2015년 Aug에 작성된 글이라서 지금이랑 뭐가 따른지 혹은 더 개선할것들은 없는지 확인해보면서 기본적인 투토리얼을 최신버전으로 정리하고자 합니다.

  1. 저자가 git에 공개한 페이지를 가져와서 하는 방법도 있지만,저자의 의도대로 mix (phoenix build tool) 로 하나하나 만드는 것을 따라해보겠습니다.

  2. 우선 준비할것들 linux or mac을 준비한상태에서 ASDF(언어 및 데이터베이스 등 개발자를 위한 설치 및 버전관리 툴)를 설치하고, ASDF를 이용해서 erlang과 elixir까지 설치 한다. https://www.icicletech.com/blog/elixir-and-erlang-setup-with-asdf-version-manager

  3. Phoenix Framework 도 설치 합니다.

    $ mix archive.install [https://github.com/phoenixframework/archives/raw/master/phoenix_new.ez](https://github.com/phoenixframework/archives/raw/master/phoenix_new.ez)

  4. 프로젝트를 만듭니다. 관련된 기본적인 dependencies를 설치 하면 세팅완료되고, 프로젝트의 소스를 git으로 관리 합니다.

    $ mix phoenix.new elixir_phoenix_blog Fetch and install dependencies? [Yn] Y

    $ cd elixir_phoenix_blog
    $ git init

  5. 데이터베이스는 기본적으로 postgres를 사용하도록 세팅되고 mysql은 설정을 변경해야합니다.

    $ vi config/dev.exs

    Configure your database

    config :elixir_phoenix_blog, ElixirPhoenixBlog.Repo,

    adapter: Ecto.Adapters.**MySQL**,
**username: "mysql_username",
password: "mysql_password",
database: "elixir_phoenix_blog",
hostname: "localhost",
port: 3306,**
pool_size: 10

    end of vi

    $ vi mix.exs # Configuration for the OTP application.

    Type mix help compile.app for more information.

    def application do
    [mod: {ElixirPhoenixCircleciBlogStarter, []}, 

    applications: [:phoenix, :phoenix_pubsub, :phoenix_html, 
:cowboy, :logger, :gettext, 
:phoenix_ecto, :**mariaex**]]

    end

    …

    defp deps do 

    [{:phoenix, "~> 1.2.1"}, 
{:phoenix_pubsub, "~> 1.0"}, 
{:phoenix_ecto, "~> 3.0"}, 
{:postgrex, ">= 0.0.0"}, 
{:phoenix_html, "~> 2.6"}, 
{:phoenix_live_reload, "~> 1.0", only: :dev}, 
{:gettext, "~> 0.11"}, 
{:cowboy, "~> 1.0"}**, 
{:mariaex, "~> 0.7"}**]

    end

    mariaex is the Ecto recommended MySQL driver.

    end of vi

    $ mix deps.get # Ecto driver 가져오기

    $ mix ecto.create

    phoenix이 CRUD을 위해 contorol,model,view(html)를 자동으로 web폴더에 생성해준다.**

    $ mix phoenix.gen.html Post posts title:string body:text**

    controller 를 라우터 파일에 넣으면 라우팅이 된다.

    $ vi web/router.ex # add resources “/posts”, PostController
    scope “/“, ElixirPhoenixCircleciBlogStarter do 

    pipe_through :browser 
# Use the default browser stack 
get "/", PageController, :index

    resources “/posts”, PostController end

    end of vi

    phoenix.gen 을 통해서 ecto.model 을 변경했으니 migrate한다 .

    $ mix ecto.migrate Compiling 8 files (.ex)
    11:35:43.432 [info] == Running ElixirPhoenixBlog.Repo.Migrations.CreatePost.change/0 forward
    11:35:43.432 [info] create table posts
    11:35:43.625 [info] == Migrated in 0.1s

    성공적!

  6. 서버를 실행시키고 첫번째 포스트를 올려봅시다.

    $ mix phoenix.server

http://localhost:4000/posts 에서 새글을 넣어본 상태

여기 까지의 파일을 올려두었으니 연습하다 다른부분이 있으면 비교해보면 좋겠습니다.
